Posted on 2018-10-30

现在动辄就在谈微服务，由于我没有认真关注，对于我而言，服务端都一样，整体的还是独立的不会有多大的影响，然而就是因为这样的想法，导致对用的东西不了解，引起了一次失误，所以还是得对自己接触的东西至少有一个宏观的了解。

一、首先了解一下什么是微服务

微服务（Microservices）是一种架构风格，一个大型复杂软件应用由一个或多个微服务组成。系统中的各个微服务可被独立部署，各个微服务之间是松耦合的。每个微服务仅关注于完成一件任务并很好地完成该任务。在所有情况下，每个任务代表着一个小的业务能力。
以往我们开发应用程序都是单体型，虽然开发和部署比较方便，但后期随着业务的不断增加，开发迭代和性能瓶颈等问题，将会困扰开发团队，微服务就是解决此问题的有效手段

认真思考了一下，我们的线上维护流程还是有很多问题的：
1、服务器一百多台，但是一挂就全线挂了 。既然用edas做的分布式应用发布，线上有几套服务，能不能一套挂了，另一套迅速补上呢？
2、数据库数据被删了，全线大面积瘫痪。数据库没有做分布式的么？只有一套可用？
3、发布不规范，一个是发布方式，二个是发布策略。常用的是class文件更换的方式。然后只要更新了服务，全部用户都会受到影响。

hexo+github搭建个人博客

Posted on 2018-10-19

一、准备github repository

1、新建一个仓库名称 repository name 设置为：用户名.github.io（eg：essencepei.github.io）
2、必须选中Initialize this repository with a README

二、准备hexo

1、在本地新建一个空文件夹
2、进入文件夹所在目录，npm install hexo -g安装hexo
3、hexo -v 验证是否安装成功
4、hexo init，看到Start blogging with Hexo！表示成功
5、npm install 安装nodejs modules
6、hexo g
7、hexo s本地预览，看到hexo页表示成功

三、连接hexo 和 github page

1、文件夹下启动git命令窗口
2、git config –global user.name “” //github用户名
git config –global user.email “” //github登录邮箱
ssh -T git@github.com 测试添加ssh是否成功，成功表示可以发布到github上了
3、修改_config.yml
deploy:
type: git
repository: https://github.com/essencepei/essencepei.github.io.git
branch: master

四、新建文章

hexo new post “文章名称”

五、本地预览

hexo clean
hexo s
预览地址：localhost:4000

六、发布到github page

hexo d -g

七、更换主题

1、git clone https://github.com/giscafer/hexo-theme-cafe.git themes/cafe 下载主题源码文件
2、修改博客配置文件 _config.yml 主题属性 theme 为 cafe.
3、cd themes/cafe
git pull
